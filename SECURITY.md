# Security policy

CAELIX is a research codebase. While it is not primarily designed as a security-hardened production service, security reports are welcomed and handled responsibly.

## Reporting a vulnerability

If you believe you have found a security vulnerability, please report it privately:

- Email: alan@caelix.co.uk

Include, where possible:

- a short description of the issue and potential impact
- the affected version/commit (or approximate date)
- steps to reproduce (ideally a minimal example)
- any suggested mitigation

Please avoid publishing details publicly until we have had a chance to assess and address the report.

## Supported versions

Security fixes will be applied to the latest development branch. If a tagged release exists, fixes may also be backported where practical.

## Acknowledgements

If you would like to be credited for a report, include the name/handle you would like to appear in the acknowledgement.
