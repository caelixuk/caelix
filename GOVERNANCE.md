

# Governance

CAELIX is maintained by Alan Ball.

The project’s goal is to provide a rigorous, reproducible research codebase for discrete-to-continuous field coupling experiments. Changes are evaluated primarily on correctness, clarity, reproducibility, and regression safety.

## Decision-making

- The Maintainer has final decision-making authority on scope, design, and merges.
- Discussion is welcome; decisions should remain evidence-led (reproductions, logs, diffs, references).
- Small, well-scoped pull requests are preferred.

## Contribution expectations

- New experiments should follow the canonical runner structure and include provenance logging.
- Changes that affect outputs should include reproduction commands and before/after artefacts.
- Performance work should include timings and validation that results are unchanged.
